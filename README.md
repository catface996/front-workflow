# 前端设计稿还原工作流

> 核心理念：**分析 → 规划 → 实现 → 校验**
> 实现阶段强调：**可视化、可沟通、可验证**

## 工作流概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  1. 分析    │ →  │  2. 规划    │ →  │  3. 实现    │ →  │  4. 校验    │
│  Analysis   │    │  Planning   │    │ Development │    │ Validation  │
├─────────────┤    ├─────────────┤    ├─────────────┤    ├─────────────┤
│ • 整体审视  │    │ • 技术选型  │    │ • 搭建骨架  │    │ • 像素对比  │
│ • 组件拆分  │    │ • 目录设计  │    │ • 布局实现  │    │ • 多端测试  │
│ • 标注提取  │    │ • 设计映射  │    │ • 样式填充  │    │ • 设计沟通  │
│ • 资源导出  │    │ • 响应式策略│    │ • 组件封装  │    │             │
│             │    │             │    │ • 交互实现  │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                             ↓
                                    ┌─────────────────┐
                                    │  可视化  Debug  │
                                    │  可沟通  确认点 │
                                    │  可验证  检查表 │
                                    └─────────────────┘
```

## 关键原则

```
不要一上来就写代码，先理解再动手
```

- **先整体后局部**：避免陷入细节
- **先静态后动态**：结构稳定后再加交互
- **保持与设计沟通**：遇到不清楚的地方及时确认

## 快速开始

1. **分析**：拿到设计稿 → 完成 [01-analysis.md](./01-analysis.md) 的确认清单
2. **规划**：确定技术方案 → 按 [02-planning.md](./02-planning.md) 建立项目结构
3. **实现**：引入debug工具 → 按 [03-implementation.md](./03-implementation.md) 可视化开发
4. **校验**：对照 [04-validation.md](./04-validation.md) 检查清单 → 交付
